create_runner(command, runner_path, template='#!/bin/bash -l\n# Name of the job - Must be unique\n#SBATCH -..., job_name='job', stdio_path='stdio.txt', stderr_path='stderr.txt', partition_name='debug', time_limit='00-06:00:00', memory_limit=512)

source code 

Create a runner (a BASH script) to run the specified job.

Parameters:
  • command - The command to execute. This should be the full path to the file that is being executed along with any command line arguments.
  • runner_path - The full path to create the runner.
  • template - The template BASH script to use. Refer to the included template, "runner_template", for more details.
  • job_name - The name of the job (should be unique).
  • stdio_path - The full path to the standard out file.
  • stderr_path - The full path to the standard error file.
  • partition_name - The name of the partition.
  • time_limit - The maximum time limit.
  • memory_limit - The maximum memory requirements in MB.
Returns:
A string containing the new BASH script.

execute_runner(runner_path, queue_limit=5000)

source code 

Execute the runner script.

Parameters:
  • runner_path - The full path to create the runner.
  • queue_limit - The max number of jobs to have in the queue. If more jobs are being launched than are able to be handled by the queue, the system will sleep, and then try again.

Variables Details [hide private]

runner_template

The runner template to use. This is a BASH script that allows you to run your jobs.

It must have the following items:

  • {0} = The name of the job
  • {1} = The full path to the standard out file
  • {2} = The full path to the standard error file
  • {3} = The name of the partition to use
  • {4} = The maximum time limit.
  • {5} = The maximum memory requirements in MB.
  • {6} = The command to execute. This should be the full path to the file that is being executed along with any command line arguments.
